Все данные хранятся только в общей БД
Создай отдельную группу защищённых эндпоинтов: /api/admin/*

Обязательная проверка ADMIN_API_KEY (секретная переменная окружения)

3. Бизнес-логика (должна полностью поддерживаться)

Два баланса: balance_usdt (реальные деньги) + virtual_balance (выигрыши). Пользователь видит сумму обоих.
При выводе сначала списывается virtual_balance, потом balance_usdt.
Торговля: результат предопределён (predetermined_result). По умолчанию 73% WIN / 27% LOSS. Выигрыш идёт на virtual_balance.
Lucky Mode, реферальная система (5%), комиссии (депозит 0%, вывод 0%, обмен 2%, торговля 2%) — всё как обычно.
При любом изменении баланса через админку → создавать запись в transactions с type = 'admin_adjust'
Все действия администратора логировать в admin_logs

4. API эндпоинты для админ-панели (/api/admin/*)
Dashboard

GET /api/admin/stats — общая статистика + количество онлайн пользователей

Пользователи

GET /api/admin/users — список с поиском и фильтрами
GET /api/admin/users/:id — полный профиль пользователя (telegram_id, username, ссылка на TG профиль)
POST /api/admin/users/:id/balance — изменить balance_usdt и/или virtual_balance
POST /api/admin/users/:id/status — изменить verified, premium, blocked + reason
POST /api/admin/users/:id/lucky — управление Lucky Mode
POST /api/admin/users/:id/winrate — установить custom_win_rate (индивидуальный шанс выигрыша)

Lucky Mode, Выводы, Рассылка, Чеки, Логи — полный набор эндпоинтов (approve/reject вывода, broadcast, создание чеков и т.д.)
